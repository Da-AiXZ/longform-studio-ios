# iOS 16.6.1 真机验收清单

记录测试日期、commit SHA、IPA SHA-256、设备型号和 TrollStore 版本。任何阻断性失败都应先修复，再更新发布包。

## 安装与数据

- `LongformStudio-TrollStore.ipa` 能被 TrollStore 安装、启动和覆盖更新
- 首次启动直接进入作品库，不出现签名、权限或启动崩溃
- 新建作品并杀掉应用后，作品、规划和正文仍存在
- 导出 `.novelproj`，删除作品，再导入备份，正文版本、审稿和事实完整恢复
- 解压或搜索工程备份，确认不存在 API Key
- 创建至少 100 章的大工程，冷启动不会一次性加载全部正文或明显卡死
- v0.1 `.novelproj` 导入后默认进入 AI 模式，原章节、版本、审稿与事实无损
- Agent 对话、待确认请求、任务范围、当前步骤和暂停位置在杀掉应用后仍可恢复

## Agent 新手流程

- 新建作品只要求作品名，创建后默认进入 Agent 对话，不出现原三步专业表单
- 未配置模型时显示单一“连接模型”引导；添加一个模型后所有角色可默认复用
- 多轮对话能生成结构化方案卡；要求修改不会写入工程，确认后自动更新设定、人物、规则、卷纲和章卡
- 模型请求未知工具、删除工程、导出或扩大执行范围时，应用拒绝执行
- AI/手动模式可以随时切换，原写作、规划、资料和质检工作区均可用
- 监督模式在执行范围和每章定稿前请求确认，中间生成与审稿自动执行
- Pass 启动前显示章节范围和最大调用次数，默认当前章；达到范围后停止
- Pass 自动准备章卡、生成、采用、四审、最多两轮修订、提取事实、门禁和批准
- 四审不完整、门禁不通过、事实冲突、两轮修订失败或达到调用上限时 Pass 暂停，不使用人工覆盖
- 切后台或锁屏后当前网络请求停止，返回应用可从上一个安全步骤恢复

## 长篇分析与模板

- 从作品库或 Agent 工具上传 UTF-8/UTF-16 TXT、Markdown 可开始流式分析
- 分析几百万字文件时界面保持响应，进度持续更新，点击停止能取消
- 分析期间内存不会按全文大小增长，完成或取消后不存在上传正文临时目录
- 本地结果包含章节统计、开头/章末模式、高频概念和关系索引
- AI 综合前显示覆盖章数和最多约 80,000 Token；没有模型时仍可保存本地统计模板
- 模板只显示抽象文风、结构、节奏、爽点、伏笔和钩子策略，不包含来源作品长句
- 全局模板可跨作品选择；应用后工程保存独立快照，修改或删除母版不改变现有作品

## 运行自检

- Agent 工具菜单可运行工程、版本引用、Agent 恢复、Keychain、磁盘、文件保护、模型配置、索引和临时目录检查
- 真实模型连接测试默认关闭；开启后明确提示会产生一次少量模型调用
- 自检结果进入 Agent 对话，并可分别导出 Markdown 和 JSON
- 报告包含应用/系统版本、检查编号、错误域、错误码、步骤、结果和建议
- 报告和持久诊断中不存在 API Key、Authorization 内容、章节正文或上传小说片段

## 编辑器

- 中文全拼、双拼或实际使用的输入法可以连续输入，不丢字、不重复提交拼音
- 输入法候选状态下自动保存不会破坏 marked text
- 长按选择、复制、粘贴、选段改写、撤销和重做正常
- 查找下一个、替换一次、全部替换正常
- 10 万字单章滚动、编辑和切换章节无不可接受卡顿
- 键盘、横竖屏、动态字体和深色模式下无控件遮挡
- 已批准版本继续编辑时会派生新版本，旧版本不变化

## AI 请求

- HTTPS 接口测试连接成功；HTTP 地址无法保存
- Key 保存后重启应用仍可使用；修改模型配置不会在界面回显 Key
- 普通和 SSE 流式接口均能得到正文
- 点击停止后请求取消，已接收文本保存为部分候选
- 生成过程中切后台或锁屏，请求取消且部分内容保留
- 401、429、500、超时和断网能显示可理解错误，不覆盖正文
- 输出达到 token 上限时，续写结果没有明显重复接缝
- 不同角色可以绑定不同 endpoint 和模型

## 创作闭环

- 创意三选一、故事圣经候选、卷纲候选和章纲二选一均只作为候选显示
- 章节生成后当前正文保持不变，候选出现在版本列表
- 段落差异正确标记插入、删除和不变段落
- 采用候选后当前版本切换，旧版本可恢复查看
- 四类审稿与本地扫描均能形成问题列表
- 选择审稿问题后，定向重写只生成新候选
- 四审未完成、分数不足或存在高等级问题时自动批准失败
- 填写足够具体的人工覆盖理由后可批准，理由保存在版本记录
- 候选事实在批准前不进入正式上下文；批准后无冲突事实入账，有冲突事实被拒绝
- 每批准十章后运行跨章回归，能生成跨章问题报告

## 导出与平台准备

- 全书 TXT、Markdown 和当前章节导出可由 Files、邮件或其他应用打开
- 中文文件名和正文编码正确
- 导出前能看到未解决问题提示
- 起点和番茄质量基线均显示“非官方规则”说明
- 投稿前人工核对平台当前规则、AI 内容标识要求、敏感内容和权利归属

## 通过标准

- 没有数据丢失、密钥泄露、正文静默覆盖或无法安装问题
- 没有导致主流程无法完成的崩溃或界面遮挡
- 一个真实项目能够从章卡走完生成、审稿、修订、批准、事实入账和导出闭环
- 一个不了解章卡、四审和事实台账的新用户只通过 Agent 对话即可完成第一章闭环
- Pass 只批准质量通过的版本，并严格在用户选择的批次范围停止
- 五百万字级文件可以本地完成有界索引，发送给模型的仅为统计、关系和少量释义证据
- 功能失败后可以生成不含密钥和正文、可直接转发的自检报告
